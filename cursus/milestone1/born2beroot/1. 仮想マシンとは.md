あるコンピュータ上で別のコンピュータを疑似的に再現したもの。

### 仮想マシンの導入目的、メリット
- 本来は複数のマシンを必要とするシステムを、マシン一台で再現できる。
- マシンが複数必要な時、仮想マシンであれば物理マシンを用意するのと比べて電力やスペースを削減できる。
- 稼働率の低い物理マシンに複数の仮想マシンを載せて、リソースを有効活用できる。

#### アーキテクチャの種類
仮想マシンは、主に
- ハイパーバイザ型（タイプ１ハイパーバイザ）
- ホスト型（タイプ２ハイパーバイザ）
- コンテナ型
の3種類の実現方法がある。

#### 1. ハイパーバイザ型（タイプ１ハイパーバイザ）
![[ハイパーバイザ型仮想化.avif]]
OSの代わりにハイパーバイザと呼ばれる仮想環境を管理するソフトウェアをインストールし、その上でOSを動かす。
ホストOS型と比べて物理サーバのOSにリソースを割く必要がないため、計算資源を有効活用できる。
- VMWare ESX-i
- Linux KVM
- Microsoft Hyper-V
- Xen

#### 2. ホストOS型（タイプ２ハイパーバイザ）
![[ホストOS型仮想化.avif]]
物理マシンにOSをインストールして、その中に仮想化ソフト（仮想マシン作成のためのアプリケーション）をインストールして、仮想マシンを実現させる方法。
仮想マシンのゲストOSが物理リソースを使うとき、ゲストOS→仮想化ソフト→ホストOS→物理サーバーの順にアクセスする。
- Oracle Virtual Box
- VMware Player

#### 3. コンテナ型
![[コンテナ型仮想化.avif]]
「コンテナ」というアプリケーションと実行環境をまとめて隔離する仕組みを利用して、OS単位ではなくアプリケーション単位で仮想化する。
ゲストOSをインストールせず、OSはホストOSをそのまま使う。コンテナ内のアプリケーションはホストOSのカーネルを利用して動いている。
- Docker
